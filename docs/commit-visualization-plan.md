# План визуализации коммитов

## Источник данных

Данные собраны из `src/lib/catalog.js` и `src/lib/project-research.js`.

### UHA-6.0

- 8 коммитов за 2 месяца
- Март 2026: 7
- Апрель 2026: 1
- Пик активности: 6 марта
- Тип работы: bootstrap, boundary-first prototype, docs, preflight, env migration

### Genius

- 10 коммитов за 2 месяца
- Март 2026: 0
- Апрель 2026: 10
- Пик активности: 5-9 апреля
- Тип работы: layout, routing, genealogy rendering, engine stabilization

### MethodA

- 15 коммитов за 2 месяца
- Март 2026: 0
- Апрель 2026: 15
- Пик активности: 8-9 апреля
- Тип работы: living document, E2E runner, config, auth, package completion

## Что плохо в текущей подаче

- Прямоугольники выглядят как статичная таблица, а не как поток работы.
- При двух месяцах сравнение превращается в сухую дихотомию, без ощущения ритма.
- У проектов разная плотность активности, но это не считывается как форма.

## Нужная визуальная идея

Коммиты лучше показывать как "движение энергии", а не как счетчики.

### Лучший вариант для текущих данных

- Один общий график с тремя линиями.
- Ось X: дни или хотя бы даты коммитов.
- Ось Y: число коммитов в день.
- Линия должна быть сглаженной, с мягкой заливкой под ней.
- Пики должны читаться как всплески работы.

### Если нужны только месяцы

- Использовать `slopegraph`.
- Это лучше баров показывает переход от марта к апрелю.
- Для каждого проекта одна линия между двумя точками.

## Рекомендованный формат

### Основной график

- Тип: area chart или smooth line chart
- Подпись: `Коммит-волна по проектам`
- Цвета:
  - UHA-6.0: янтарный
  - Genius: синий
  - MethodA: зелёный
- Линии должны быть тонкими, но с glow-эффектом.
- Подсветить только ключевые пики и итоговое количество коммитов.

### Дополнительные элементы

- Справа от графика показать итоговый объём: `8`, `10`, `15`.
- Ниже дать короткий текстовый вывод:
  - где работа была ранней и короткой,
  - где был резкий апрельский всплеск,
  - где проект шел как длительный документальный поток.

## Диалектический анализ формы

- UHA-6.0: ранний спринт с затуханием. Лучше показывать как короткую волну, которая быстро уходит вниз.
- Genius: апрельский подъём. Нужна крутая линия роста с несколькими пиками.
- MethodA: наиболее плотный и цельный поток. Лучше всего выглядит как высокая устойчивая волна.

## Практическая схема экрана

1. Заголовок: `Коммит-волны`
2. Краткий вывод: кто работал рывком, кто серией, кто непрерывно
3. Основной line/area chart
4. Мини-легенда с итогами по проектам
5. Под графиком пояснение по датам и смыслу пиков

## Что нужно сделать в коде

- Убрать разрозненные прямоугольники из блока активности.
- Построить один совместный график по проектам.
- Если есть дневная детализация, агрегировать коммиты по дням.
- Если дневной детализации нет, строить slopegraph по месяцам.
- Добавить плавность кривой и мягкую заливку.
- Добавить метки только на пиках.

## Итог

Для этой задачи лучший язык визуализации - не таблица и не столбцы, а волна. Она показывает ритм, плотность и направление работы одновременно.
